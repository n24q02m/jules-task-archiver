## 2025-02-23 - Prevent DoS via Storage Exhaustion in Background Script
**Vulnerability:** The `CACHE_START_CONFIG` message handler in the background script did not validate the size of the `msg.config` payload before storing it in `chrome.storage.session`, which has a strict quota (e.g., 1MB or 10MB limit). A malicious or malfunctioning content script could send massive payloads, quickly exhausting the storage quota and causing a Denial of Service (DoS) for the extension.
**Learning:** Even internal messages between trusted components (like content scripts and the background worker) should be validated. `chrome.storage` quotas are strict and exceeding them causes unhandled exceptions that can break extension functionality. Bounding payload size is crucial for robustness.
**Prevention:** Always validate and bound the size of payloads received via `postMessage` or `chrome.runtime.sendMessage` before attempting to persist them to `chrome.storage`, particularly when dealing with session or local storage with low limits. We introduced a 50KB JSON stringification limit check.
